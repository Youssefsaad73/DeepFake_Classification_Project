TME-6017: Final Project Instructions
Due Date: -----
Weight: 25 marks 


Project Description
In this final project, students will apply deep learning techniques to address a real-world problem.
The students will construct their own dataset, leverage pre-trained models for fine-tuning, or train a new model from scratch.
The project should show the application of classification and autoencoders.
The end solution must be integrated with a downstream application or process that demonstrates the practical value of the machine learning system.
The project should clearly articulate the business or societal value of the solution, in line with the problem statement provided at the project’s inception.


Final Project Evaluation Criteria:
Introduction (1 mark)
Problem Statement
Define the specific machine learning problem.
Explain the importance of solving this problem.
Business Value
Describe the potential impact on industry or society.
Identify the stakeholders and articulate the benefits to them.

Analysis of dataset Bias (5 marks)

From Assignment 4, discuss bias identified in dataset selected
Discuss bias mitigation strategies for your own dataset
Apply bias mitigation strategies
Generate preprocessed dataset

Methodology (9 marks)
Literature Review (2 marks)
Summarize relevant research and existing solutions.
Identify gaps that your project aims to address.
Model Selection and Justification (2 marks)
Explain the choice of pre-trained model or the rationale for a new model.
Discuss the expected advantages of the selected approach.
Data Preparation using OpenCV (2 marks)
Detail the data collection, augmentation, and preprocessing steps.
Describe the dataset split for training, validation, and testing.
Training and Fine-tuning (2 marks)
Describe the training process, including any fine-tuning of pre-trained models.
Discuss the choice of hyperparameters and the tuning strategy.
Integration with Downstream Pipeline (1 mark)
Explain how the model's output will be utilized in a downstream application.
Detail the expected workflow and how the solution adds value.

Implementation and Results (9 marks)
Code and Repository (6 marks)
Provide clean, well-documented code in a public repository.
Include instructions for setup, training, and inference.
Model Performance (2 marks)
Present results, including accuracy, precision, recall, and other relevant metrics.
Provide visualizations such as confusion matrices, ROC curves, or sample outputs.
Downstream Application Integration (1 mark)
Demonstrate the integration with a downstream application or process.
Show how the model's output is used to provide actionable insights or solve the problem.

Discussion and Conclusion (1 mark)
Interpretation of Results
Limitations and Future Work
Acknowledge any limitations of the current approach.
Suggest potential improvements or areas for future research
